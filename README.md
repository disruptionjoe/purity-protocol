# Purity Protocol

An open-source effort to make decentralized decision-making durable by
building a corruption-, collusion-, and capture-resistant substrate for
rewarding informed participation.

## Current state

Purity Protocol is in public, founder-led Phase 0 formation. It is a project to
research and govern a protocol, not a live protocol. Joe is currently the only
person eligible to contribute effort, make decisions, or change governance
rules.

There is no live application, pilot, participant program, incentive system,
treasury, deployment, or production mechanism. Public visibility and an MIT
code license do not open participation or imply that any proposed mechanism
works.

## What the project is doing

Phase 0 is defining the initial conditions for an evolving institution:
constitutional authority, three distinct forms of eligibility, rule-change
processes, meaningful consent and exit, real reversibility, an
evidence-gated path for progressively transferring authority, and the research
needed before a first expansion decision can responsibly be made.

The repository's inward-facing responsibility is:

> Purity Protocol is a sovereign open-source project for designing and
> governing rules that make recurring, opt-in participation produce better,
> correctable collective signal while progressively expanding authority
> without sacrificing coherence or capture resistance.

## Claims and participation

Pairwise comparison, confidence elicitation, peer prediction, system-native
issuance, and related mechanism ideas are research hypotheses. They are not
constitutional requirements or proven solutions.

Outside contribution is not yet open. Phase 0 must first produce a legible
eligibility-expansion gate, and Joe must separately decide whether that gate is
satisfied. The unresolved contribution, documentation-license, patent, and
portable-creation questions are tracked in [RESEARCH.md](RESEARCH.md).
